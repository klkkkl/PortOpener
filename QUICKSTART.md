# 快速入门指南

## 🚀 5分钟上手 PortOpener

### 第一步：启动应用

```bash
# 开发模式
pnpm tauri dev

# 或使用已构建的应用程序
# Windows: 双击 PortOpener.exe
# macOS: 打开 PortOpener.app
# Linux: ./portopener
```

### 第二步：添加第一个转发规则

#### 场景 1: 转发本地 HTTP 服务

假设你有一个运行在 `localhost:3000` 的开发服务器，想让局域网内其他设备访问：

1. 点击 **"+ Add Rule"** 按钮
2. 填写表单：
   - **Name**: `Dev Server`
   - **Protocol**: `TCP`
   - **Listen Address**: `0.0.0.0:8080`
   - **Target Address**: `127.0.0.1:3000`
3. 点击 **"Add"** 保存
4. 点击规则行的 **"Start"** 按钮启动转发

现在局域网内的设备可以通过 `http://你的IP:8080` 访问你的开发服务器了！

#### 场景 2: SSH 端口转发

将远程服务器的 SSH 端口转发到本地：

1. 添加规则：
   - **Name**: `SSH Tunnel`
   - **Protocol**: `TCP`
   - **Listen Address**: `127.0.0.1:2222`
   - **Target Address**: `remote-server.com:22`
2. 启动规则
3. 使用 `ssh user@localhost -p 2222` 连接

#### 场景 3: 游戏服务器 UDP 转发

转发游戏服务器的 UDP 端口：

1. 添加规则：
   - **Name**: `Game Server`
   - **Protocol**: `UDP`
   - **Listen Address**: `0.0.0.0:27015`
   - **Target Address**: `game-server.com:27015`
2. 启动规则
3. 玩家可以通过你的 IP:27015 连接游戏服务器

### 第三步：管理规则

#### 停止转发
- 点击规则行的 **"Stop"** 按钮
- 状态会从 "Running" 变为 "Stopped"

#### 删除规则
- 确保规则已停止（状态为 "Stopped"）
- 点击 **"Delete"** 按钮
- 规则会从列表中移除

#### 查看状态
- **绿色圆点 + "Running"**: 转发正在运行
- **灰色圆点 + "Stopped"**: 转发已停止
- **红色圆点 + "Error: ..."**: 转发出错（查看错误信息）

### 第四步：使用技巧

#### 键盘快捷键
- **ESC**: 关闭添加规则弹窗
- **Enter**: 在表单中按 Enter 快速提交

#### 地址格式
- 监听所有网卡: `0.0.0.0:端口`
- 仅本地访问: `127.0.0.1:端口`
- 指定网卡: `192.168.1.100:端口`
- 域名: `example.com:端口`

#### 常见端口
- HTTP: 80, 8080, 3000
- HTTPS: 443, 8443
- SSH: 22
- MySQL: 3306
- PostgreSQL: 5432
- Redis: 6379
- MongoDB: 27017

### 第五步：查看日志

在开发模式下，控制台会显示连接日志：

```
[TCP] Listening on 0.0.0.0:8080
[TCP] New connection from 192.168.1.100:54321 -> 127.0.0.1:3000
[UDP] Listening on 0.0.0.0:27015
[UDP] Cleaning up idle session: 192.168.1.100:12345
```

## ⚠️ 注意事项

### 安全提示
1. **不要暴露敏感服务**: 避免将数据库、管理后台等直接暴露到公网
2. **使用防火墙**: 配合系统防火墙限制访问来源
3. **监控流量**: 定期检查转发规则，避免被滥用
4. **及时停止**: 不使用时及时停止转发规则

### 性能提示
1. **UDP 会话**: 空闲 60 秒后自动清理，无需担心内存泄漏
2. **并发连接**: TCP 支持高并发，但受系统资源限制
3. **大文件传输**: 适合流式传输，不会占用大量内存
4. **网络延迟**: 转发会增加少量延迟（通常 < 1ms）

### 故障排查

#### 规则启动失败
- **端口被占用**: 检查是否有其他程序使用该端口
  ```bash
  # Windows
  netstat -ano | findstr :8080

  # Linux/macOS
  lsof -i :8080
  ```
- **权限不足**: 监听 1024 以下端口需要管理员权限
- **地址格式错误**: 确保格式为 `host:port`

#### 无法连接目标
- **目标不可达**: 检查目标地址是否正确，网络是否连通
  ```bash
  ping target-host
  telnet target-host port
  ```
- **防火墙阻止**: 检查目标服务器防火墙设置
- **DNS 解析失败**: 尝试使用 IP 地址代替域名

#### 规则无法删除
- **规则运行中**: 必须先停止规则才能删除
- **点击 Stop 按钮**: 等待状态变为 "Stopped"
